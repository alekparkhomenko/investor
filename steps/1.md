**Задача: Реализовать Data Ingestor сервис на Go для получения котировок с Московской биржи (ISS API)**

Разработать backend-компонент (Data Ingestor), который получает рыночные данные по акциям с ISS API Московская биржа и передает их дальше в систему обработки алертов.

### Функциональные требования:

1. Реализовать polling-механизм для получения котировок через HTTP API:

   * Endpoint:
     `https://iss.moex.com/iss/engines/stock/markets/shares/securities/{SYMBOL}.json`
   * Интервал опроса: 1–2 секунды

2. Парсить ответ ISS API:

   * Обработать формат `columns + data`
   * Извлекать:

     * тикер (SECID)
     * последнюю цену (LAST)

3. Преобразовать данные в внутреннюю модель:

   ```go
   type Quote struct {
       Symbol string
       Price  float64
       Time   int64
   }
   ```

4. Передавать котировки в канал (`chan Quote`) для дальнейшей обработки.

---

### Нефункциональные требования:

1. **Concurrency:**

   * Использовать goroutines для асинхронного получения данных
   * Использовать channels для передачи данных

2. **Управление жизненным циклом:**

   * Использовать `context.Context` для graceful shutdown
   * Корректно завершать работу при отмене контекста

3. **Отказоустойчивость:**

   * Обрабатывать ошибки HTTP-запросов
   * Реализовать retry (через цикл polling)
   * Установить timeout для HTTP-клиента

4. **Производительность:**

   * Не блокировать основной поток
   * Использовать буферизированный канал

5. **Логирование:**

   * Логировать ошибки и полученные котировки

---

### Ограничения:

* Не использовать внешние брокеры сообщений (например, Apache Kafka)
* Не использовать базы данных (например, PostgreSQL или Redis)
* Весь функционал реализуется в рамках одного сервиса (MVP)

---

### Ожидаемый результат:

* Рабочий Go-сервис, который:

  * регулярно получает котировки с ISS API
  * корректно парсит данные
  * отправляет их в канал для дальнейшей обработки
  * устойчив к ошибкам сети
  * корректно завершается при остановке приложения

---

* Добавить поддержку нескольких тикеров
* Написать unit-тесты для парсинга ответа ISS API
